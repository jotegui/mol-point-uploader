Frequently Asked Questions (FAQ)

How should I prepare the file for upload?

The file must have one of the following extensions: CSV, TXT or TSV.The first row must contain the headers, and the rest of the file must contain a single record per row (with newline characters separating rows) and fields must be separated with one of the following characters (only one in a file): semi-colon (;), comma (,) or tab. Here’s an example of a comma separated file:

File Name: myDataset.csv 
Header1,Header2,Header3
Record1Field1,Record1Field2,Record1Field3
Record2Field1,Record2Field2,Record2Field3

What are the mandatory fields a file must have?

None of the fields is purely mandatory, in the sense that you can specify default values (values that will be applied to all records) for every field. However, there is a set of fields that must either be present in the file or have a default value: scientific name, latitude, longitude, coordinate uncertainty, observation date and observer name or id.

What is the best format for the mandatory fields?

In order to maximise the integration with the rest of the MOL infrastructure, we recommend you to prepare the mandatory fields in the way described below:

  • Scientific name should have a properly built binomial for the taxon
  • Coordinates should be stored in decimal degree format
  • Uncertainty should be stored as a single number, representing meters
  • Observation date and time should comply with the ISO 8601 standard

My file does not have one (or more) of the mandatory fields, but a common value can be applied to all records. Do I have to create a column just for that?

No, you don't have to! You can specify that value in the corresponding “default” field. The tool will create the column for you, populating every record with the specified value.

Some of my records are missing a value in a mandatory field because a default value can be applied. Do I have to fill these empty fields?

No, you don't have to! You can put that default value in the “default” field and it will be applied only to empty fields. It will not overwrite any existing information, just fill the blanks.

After aligning the headers, I have been redirected to the main page with a warning message. What’s wrong?

That means the parser found quality issues in your dataset. There are many possible causes, and you will be provided a link with a list of all the problems in your record/s. Most likely, it will be a type mismatch (e.g., putting letters in numeric fields like uncertainty, or putting coordinates in formats other than decimal degrees). You will need to fix those to be able to continue. See the FAQ “What is the best format for the mandatory fields?” for information on how to prepare the content of the fields.

What are the mandatory fields in the metadata form?

Just a few of them, and all have the word “Required” next to them. We encourage you to fill all fields, but only these are actually required:

  • Title of the dataset
  • Abstract of the dataset
  • Creator of the dataset
  • Creator of the metadata
  • Geodetic Datum (actually, a default value of WGS84 is applied if you leave this empty)

I have filled every form with correct values but I keep getting an “Opps! There seems to be an issue with your request” error after sending the metadata form. What’s going on?

Lately we have been experiencing difficulties with a (different) third-party service that registers the uploaded information. Unfortunately, if you have filled everything correctly and still receive this error, your data could not be uploaded and you will have to wait a few minutes and try again. If after waiting a couple hours it still doesn't work, please send us an email! You can find contact information on the team page. We are really sorry for this inconvenience.

The upload went smoothly, but I cannot see a map of the records. What's wrong?

Visualizing the uploaded points on a map is a heavy process that gets heavier as the dataset grows. Most of the times it is just a matter of patience, but there might be some cases where there is actually something wrong. Check the uploaded records for quality issues (mainly strange characters in the scientific name field) and correct any suspicious value. Then, delete and re-upload the dataset. If it is not fixed and you are sure your data are right, please send us an email with the URL to your dataset map and we will contact you shortly.
